Informasi lengkap mengenai dataset, model machine learning, serta dokumentasi API pengembang

Dokumentasi API
Selamat datang di dokumentasi API RailAnalytics. Antarmuka API ini dapat digunakan oleh pengembang untuk mengambil data analitik kereta NJ Transit secara terprogram. Semua respons dikembalikan dalam format application/json.
GET /api/kpi
Mengembalikan metrik KPI utama dari ringkasan data perjalanan NJ Transit.
Contoh Respons:
{
  "avg_delay": "3.55 min",
  "cancellation_rate": "1.99%",
  "on_time_rate": "89.7%",
  "total_lines": "9",
  "total_trips": "256,508",
  "worst_line": "Gladstone Branch"
}
GET /api/status-distribution
Mengembalikan pembagian total perjalanan berdasarkan status akhir kereta.
Contoh Respons:
[
  { "status": "departed", "total_trips": 230066 },
  { "status": "estimated", "total_trips": 21332 },
  { "status": "cancelled", "total_trips": 5110 }
]
POST /predict
Memprediksi status kereta (departed, estimated, cancelled) berdasarkan parameter perjalanan. Mendukung format JSON.
Parameter Body (JSON):
{
  "from_id": "105",
  "to_id": "38187",
  "line": "Morristown Line",
  "type": "NJ Transit",
  "stop_sequence": 5,
  "is_weekend": 0
}
Contoh Respons:
{
  "success": true,
  "result": {
    "label": "departed",
    "confidence": 92.4,
    "probabilities": {
      "departed": 92.4,
      "estimated": 6.1,
      "cancelled": 1.5
    }
  }
}
Dataset & Informasi Proyek
Dataset NJ Transit + Amtrak Rail Performance (Maret 2018) bersumber dari GitHub Tesyaf. Memiliki total data sebanyak 256.508 baris perjalanan kereta dengan fitur utama meliputi: tanggal, nomor kereta, nama jalur kereta, stasiun asal dan tujuan, jadwal waktu, waktu aktual keberangkatan, menit keterlambatan (delay), dan status operasional kereta.
Model Machine Learning
Model prediksi dibangun menggunakan pustaka Python dengan algoritma Random Forest Classifier dan rangkaian pipeline pra-pemrosesan data:
  • Imputasi nilai kosong (missing values) menggunakan metode median.
  • Standardisasi fitur numerik melalui StandardScaler.
  • Penanganan ketidakseimbangan kelas (class imbalance) menggunakan teknik SMOTE oversampling digabung dengan RandomUnderSampler.
  • Seleksi fitur bermakna menggunakan SelectKBest (k=15).
  • Konfigurasi RandomForestClassifier dengan pembobotan kelas subsampel seimbang (class_weight='balanced_subsample').
